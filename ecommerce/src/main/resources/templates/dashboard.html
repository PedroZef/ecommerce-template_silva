O que é o Controle de Transações ACID? (Aulas 1, 2 e 3)

Nesta primeira sprint do e-commerce, implementamos regras do **Spring Data JPA** e **Spring MVC** integrados, estruturando uma transação segura com o banco de dados. Abaixo explicamos as propriedades **ACID** aplicadas ao nosso código:

A Atomicidade (Tudo ou Nada)

Ao finalizar um pedido, o estoque de múltiplos produtos é reduzido. Se a quantidade de um único produto for insuficiente, a transação inteira falha e é revertida (Rollback). Nada é gravado e nenhum estoque é deduzido.

C Consistência (Regras de Integridade)

Garante que o banco vá apenas de um estado válido para outro válido. As validações JPA/Hibernate (como e-mail único, estoque positivo, chaves estrangeiras de categoria obrigatórias) são preservadas e respeitadas.

I Isolamento (Concorrência Segura)

Operações simultâneas de compra ocorrem de forma isolada sem interferir umas nas outras. O Spring e o JPA gerenciam o nível de isolamento de transações, impedindo leituras sujas de dados parciais.

D Durabilidade (Persistência Permanente)

Uma vez que a transação é finalizada com sucesso (Commit), todas as modificações (estoques reduzidos, faturas e registros de pedidos) tornam-se permanentes e não serão perdidas, mesmo em quedas do sistema.

Arquitetura de Banco de Dados Multi-Profile (H2 e MySQL)

Este projeto está configurado de forma inteligente com **Spring Profiles** para facilitar o desenvolvimento:

  • Perfil `dev` (H2 em memória - Ativo): O banco de dados roda 100% em memória, sendo limpo e recriado automaticamente no startup. A classe DataLoader carrega automaticamente categorias, produtos e clientes pré-cadastrados para uso imediato dos alunos.
  • H2 Console: Acesse localmente pelo navegador em http://localhost:8080/h2-console usando a JDBC URL jdbc:h2:mem:ecommerce_db e usuário sa (senha vazia) para visualizar as tabelas do JPA em tempo real.
  • Perfil `prod` (MySQL de produção): Para mudar para MySQL, basta ativar o profile `prod` passando o argumento --spring.profiles.active=prod.
