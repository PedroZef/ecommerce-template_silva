Instruções de Aula Prática:
  1. Caso 1 (COMMIT de Sucesso): Monte uma cesta com quantidades dentro do estoque disponível. A transação será finalizada, os estoques reduzidos no banco e você será direcionado para o Histórico de Pedidos com a mensagem de confirmação.
  2. Caso 2 (ROLLBACK de Falha): Adicione uma quantidade maior do que o estoque de qualquer produto (Ex: se há 10 unidades de Smartphone, solicite 12). O banco de dados iniciará a compra e cancelará o processo ao perceber o limite excedido, restabelecendo o estado anterior integralmente.

Comportamento ACID esperado:

  • Atomicidade: O pedido só é salvo se TODOS os produtos forem deduzidos com sucesso.
  • Consistência: Os estoques não podem ficar negativos no banco de dados.
  • Durabilidade: Pedidos bem-sucedidos persistem mesmo se o servidor for desligado.
